注释规范
---
对重要阶段与步骤添加`空行`+`行首`注释！！！


### 注释规范
#### 须知要求
- 注释简洁高效，注释率保持在30%
- 属性字段：普遍采用同行末尾注释即可，无需新增行注释（会使代码臃肿）
- 务必确保主要属性字段都有规范注释！！！！


#### 文件头注释规范
- 重要核心模块文件应在文件头部添加注释
- 主要内容：文件名、功能描述、作者、创建时间
- 注释参考示例如下
```python
"""
功能描述：xxxx（职责描述）
作者：qdbin
"""
```


#### 常量/枚举注释规范
- 常量和枚举值采用同行末尾注释
- 注释应说明该值的含义和使用场景
- 注释参考示例如下
```python
# 订单状态枚举
class OrderStatus(Enum):
    PENDING = "pending"      # 待处理状态，可进行操作
    PROCESSING = "processing"  # 处理中，禁止重复操作
    COMPLETED = "completed"  # 已完成，最终状态
    CANCELLED = "cancelled"  # 已取消，可再次下单

# 系统配置常量
MAX_RETRY_COUNT = 3         # 最大重试次数
DEFAULT_TIMEOUT = 30        # 默认超时时间（秒）
```


#### 接口/抽象类注释规范
- 采用多行注释
- 主要侧重内容：职责定义、子类需实现的方法、调用约束
- 注释参考示例如下
```python
class BaseParser(ABC):
    """
    解析器基类，定义解析器的通用接口
    子类必须实现parse方法，处理具体解析逻辑
    """
    @abstractmethod
    def parse(self, content: str) -> Dict:
        """
        解析内容为结构化数据
        @param content: 待解析的原始内容
        @return: 解析后的结构化字典
        @throws ParseError: 解析失败时抛出
        """
        pass
```


#### 类注释规范
- 采用多行注释
- 主要侧重内容：职责简述
- 注释参考示例如下（仅供参考）
```python
class xx:
  """
  xxx(类名翻译：简单介绍)
  xxx（其他额外描述）
  """
```

#### 方法函数注释
- 采用多行注释
- 注释侧重：函数功能，参数，返回值、实现阶段及步骤
- 方法步骤实现注释
  - 对方法步骤实现进行规范简洁且精要的注释
  - 常规步骤在其同行末尾添加简单注释
  - “关键分支及步骤阶段”需在其上方新增行，添加独行注释
  - 注释语言简洁专业，清晰明了，说明代码本质功能
  - 适当添加空行，以便我浏览
- 部分复杂函数功能要有演示示例（选有）（侧重输入，输出示例）
- 注释参考示例如下（仅供参考）
  ```python
  def xxx(xxx)
    """
    xxx（函数说明）
    @param xxx: xxx（简单说明）
    @return: xxx （简单说明）

    示例：……（选有，复制函数的重要注释说明）
    """
    // xxx（独行注释，重要阶段）
    code………………  // xx（同行末尾注释，个别小步骤难点）
    [空行]
    // xxx（独行注释）
    code……………… 
    code………………  // xx（同行末尾注释）
  ```

#### 其他规范
- 对特别复杂方法可额外添加实现说明，合理缩进分层，保持内容简洁
- 对较复杂业务方法的示例，1个即可，较为复杂的2个左右（即严格控制数量，不要太多）
- 严格限定于注释内容，即不要包含以下相关的繁琐内容（不局限于，这样会使源码及其臃肿）：
  - 技术实现细节
  - 执行流程描述
  - 设计思路
  - 无关技术术语
  - 项目介绍
